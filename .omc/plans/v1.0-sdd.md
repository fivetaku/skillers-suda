# skillers-suda v1.0 SDD (Software Design Document)

## 목표

skill-creator(Anthropic)의 검증/최적화 기능을 통합하여 **생성→검증→최적화→패키징** 원스톱 스킬 빌더로 업그레이드.

## 아키텍처 변경

```
v0.7.0 (현재)                    v1.0.0 (목표)
Phase A: 아이디어 수집             Phase A: 아이디어 수집 + 대화 추출
Phase B: 4인 팀 토론              Phase B: 4인 팀 토론 + MCP 추천
Phase C: 상세 인터뷰              Phase C: 상세 인터뷰
Phase D: 워크플로우 설계           Phase D: 워크플로우 + eval 설계
Phase E: 파일 생성                Phase E: 파일 생성
Phase E-verify: 구조 검증         Phase E-verify: 구조 검증 (강화)
Phase F: 수동 테스트              Phase F: 자동 eval 실행 + 벤치마크
                                 Phase G: 반복 개선
                                 Phase H: Description 최적화
                                 Phase I: 패키징
```

### Task Group 2: SKILL.md 앞단 개선 (순차 작업)

#### Task 2-1: Phase A 개선 — 대화 컨텍스트 추출
- 변경: 기존 대화에서 워크플로우 추출 패턴 추가
- 위치: Phase A 상단

#### Task 2-2: Phase B 개선 — MCP 추천
- 변경: 전문가 에이전트 프롬프트에 "6. 추천 MCP" 항목 추가
- 참조: references/mcp-catalog.md
- 위치: 전문가 에이전트 프롬프트

#### Task 2-3: Phase D 개선 — 현실적 테스트 설계
- 변경: eval 시나리오를 evals/evals.json 형식으로 강화
- 추가: should-trigger / should-not-trigger 구분, 현실적 프롬프트 철학
- 위치: Phase D eval 기준 정의

#### Task 2-4: Phase E 개선 — Writing 철학 업데이트
- 변경: Pushy description, Explain the why, 트리거 메커니즘 설명 추가
- 제거: v0.7.0 description 후보 비교 (run_loop.py로 대체)
- 위치: Phase E Writing Style 규칙

### Task Group 3: SKILL.md 뒷단 재작성 (순차 작업, Group 1 완료 후)

#### Task 3-1: Phase F 재작성 — 자동 eval 실행 + 벤치마크
- 기존 Phase F 전면 교체
- 내용: evals.json 생성 → subagent 실행 → assertion → grading → benchmark → viewer
- 참조: skill-creator SKILL.md의 "Running and evaluating test cases" 섹션

#### Task 3-2: Phase G 신설 — 반복 개선
- 내용: 피드백 기반 수정 + 일반화 원칙 + 반복 코드 번들 + iteration 관리
- 참조: skill-creator의 "Improving the skill" 섹션

#### Task 3-3: Phase H 신설 — Description 최적화
- 내용: 20개 eval 쿼리 → eval_review.html 리뷰 → run_loop.py 실행 → best_description 적용
- 참조: skill-creator의 "Description Optimization" 섹션

#### Task 3-4: Phase I 신설 — 패키징
- 내용: package_skill.py → .skill 파일 생성

#### Task 3-5: AI 행동 규칙 업데이트
- 추가: eval 자동 실행, description 최적화, 일반화 원칙, 반복 코드 번들
- 제거: v0.7.0 description 후보 비교 관련 규칙

#### Task 3-6: References 섹션 업데이트
- 추가: schemas.md, agents/grader.md, agents/comparator.md, agents/analyzer.md, mcp-catalog.md
- 추가: improvement-principles.md, trigger-mechanism.md (신규 생성)

### Task Group 4: 신규 reference 파일 생성 (독립, 병렬 가능)

#### Task 4-1: references/improvement-principles.md
- 내용: 일반화 원칙, lean 유지, why 설명, 반복 코드 번들 (skill-creator 철학)

#### Task 4-2: references/trigger-mechanism.md
- 내용: 스킬 트리거 메커니즘, undertrigger 경향, pushy 전략, 복잡한 쿼리만 트리거

### Task Group 5: 마무리 (순차)

#### Task 5-1: 기존 코드 정리
- v0.7.0 description 최적화 (후보 2-3개) 섹션 제거
- verify-skill.py + quick_validate.py 기능 병합

#### Task 5-2: 메타데이터 업데이트
- plugin.json: version 0.7.0 → 1.0.0
- CHANGELOG.md: v1.0.0 엔트리 추가

#### Task 5-3: 자체 검증
- verify-skill.py 실행
- 단어 수 확인 (SKILL.md가 많이 커질 예정 — references 분리 필요할 수 있음)

#### Task 5-4: 커밋 + 배포
- 서브모듈 커밋 + 푸시
- 부모 저장소 업데이트
- 마켓플레이스 동기화 + 캐시 교체

## 병렬화 전략

```
[병렬 Wave 1] Task 1-1 ~ 1-5 + Task 4-1 ~ 4-2 (파일 복사 + 신규 reference)
     ↓ 완료 대기
[순차 Wave 2] Task 2-1 ~ 2-4 (SKILL.md 앞단 — 하나의 에이전트가 순차 수정)
     ↓
[순차 Wave 3] Task 3-1 ~ 3-6 (SKILL.md 뒷단 — 하나의 에이전트가 순차 수정)
     ↓
[순차 Wave 4] Task 5-1 ~ 5-4 (마무리)
```

## 의존성

- Task Group 3은 Task Group 1 완료 후 시작 (복사된 파일 참조 필요)
- Task Group 5는 모든 Task 완료 후 시작
- Task Group 1, 2, 4는 서로 독립 → 병렬 가능
